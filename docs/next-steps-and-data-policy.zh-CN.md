# ScreenLingua 后续任务与数据规范

## 后续任务清单

### P0：真机运行前置环境

- 安装 App Installer / winget，或手动安装后续工具。
- 安装 Rustup，确保 `rustc` 和 `cargo` 可用。
- 安装 Visual Studio Build Tools，并选择 C++ build tools。
- 重新打开 PowerShell 后运行：

```powershell
pnpm env:check
```

验收标准：`pnpm env:check` 全部通过，`pnpm tauri:dev` 能启动桌面客户端。

### P1：桌面客户端真机测试

- 运行：

```powershell
pnpm tauri:dev
pnpm test:real-machine
```

- 测试窗口打开、侧边栏切换、深色模式、设置页服务状态、清除缓存。
- 测试固定区域截图、OCR、翻译、缓存命中。
- 用真实英文截图重新跑：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\test_local_flow.ps1 -Image "你的截图路径.png"
```

验收标准：RapidOCR 和 Argos 都走真实本地引擎，截图内容能识别并翻译，重复测试能命中缓存。

### P1：客户端 UI 完善

- 做真正的区域选择覆盖层：拖拽框选、显示坐标、支持确认/取消。
- 补全固定区域管理：新增、重命名、删除、启用/暂停。
- 历史页接真实 SQLite 数据，不再只用 mock 数据。
- 术语库支持搜索、导入、导出、恢复默认。
- 设置页补充模型状态、数据目录、日志目录、缓存容量和清理按钮。

验收标准：用户不需要命令行，也能完成截图翻译、术语维护、缓存清理和问题诊断。

### P2：数据边界实现

- 增加统一路径管理模块，所有运行数据必须从同一个模块取路径。
- 增加数据目录体检：启动时检查实际写入路径是否符合本规范。
- 增加容量和保留期策略：截图临时文件、日志、SQLite 历史、缓存都要有上限。
- 增加“导出诊断包”功能，只打包允许导出的日志和环境信息，不包含截图原图、OCR 原文、隐私黑名单命中内容。

验收标准：代码里不再散落随手拼接的数据路径；任何新数据落盘点都能在本文件登记。

### P2：打包与安装

- 跑通：

```powershell
pnpm tauri:build
```

- 安装生成的 Windows 包。
- 卸载后检查是否留下大体积残留。
- 验证安装版和开发版不会互相污染数据。

验收标准：安装版可启动、可 OCR/翻译、可清理缓存，卸载后没有不可解释的数据残留。

### P3：质量与体验

- 增加前端组件测试或 Playwright 冒烟测试。
- 增加 sidecar API 回归测试：健康检查、术语保存、OCR、翻译、缓存清理。
- 增加模型缺失、端口占用、Python/Rust 缺失时的用户友好提示。
- 清理 README 中旧目录路径，把示例统一成当前项目路径。

## 数据放置规范

原则：源码归源码，配置归配置，运行数据归运行数据；任何数据写入都必须有明确目录、容量上限和清理方式。

### 允许的数据位置

| 数据类型 | 允许位置 | 说明 |
| --- | --- | --- |
| 源码、脚本、文档 | 项目目录 | 只放可版本化的内容，例如 `src`、`src-tauri`、`scripts`、`docs` |
| 默认术语表 | `config/glossary.zh-CN.json` | 这是项目配置，可以随项目维护 |
| 开发日志 | `logs/` | 仅开发期使用，必须在 `.gitignore` 中忽略 |
| 前端构建产物 | `dist/` | 构建输出，可删除、可重建 |
| Node 依赖 | `node_modules/` | 包管理器生成，可删除、可重建 |
| sidecar 虚拟环境 | `sidecars/ocr_service/.venv/` | Python 依赖环境，可删除、可重建 |
| 模型文件 | `%LOCALAPPDATA%\ScreenLingua\models` | Argos、OCR 等本地模型，体积大，不进项目目录 |
| 桌面端 SQLite 数据库 | `%LOCALAPPDATA%\ScreenLingua\screenlingua.db` | 翻译历史、缓存、术语运行态数据 |
| 桌面端日志 | `%LOCALAPPDATA%\ScreenLingua\logs` | 安装版运行日志，按大小/天数滚动 |
| 临时截图 | `%TEMP%\ScreenLingua\captures` | 仅用于一次 OCR 请求，处理后应自动清理 |

### 禁止行为

- 禁止把模型、截图、数据库、大日志写到项目根目录。
- 禁止默认写入 `D:\`、桌面、下载目录、用户文档根目录等无明确归属的位置。
- 禁止把用户截图原图长期保存，除非用户明确开启“保留截图历史”。
- 禁止把 OCR 原文、翻译历史、截图路径写进普通调试日志。
- 禁止新增未登记的数据目录。新增前必须更新本规范。

### 容量和清理要求

- 临时截图：默认请求结束后删除；异常退出时，下次启动清理超过 24 小时的文件。
- 开发日志：单文件建议不超过 5 MB，保留最近 5 个。
- 桌面端日志：按天或大小滚动，默认保留 14 天。
- SQLite 翻译缓存：需要提供最大条数或最大数据库大小配置。
- OCR/翻译内存缓存：必须有上限，当前由 `SCREENLINGUA_OCR_CACHE_MAX` 和 `SCREENLINGUA_TRANSLATION_CACHE_MAX` 控制。
- 模型目录：不自动删除模型，但设置页必须显示路径和占用空间。

### 新增数据写入点的检查清单

- 这个数据是否必须落盘？能否只放内存？
- 是否包含截图、OCR 原文、翻译历史、路径、窗口标题等敏感信息？
- 写入目录是否在“允许的数据位置”表中？
- 是否有容量上限、保留期、清理入口？
- 是否会被 Git、诊断包、日志收集脚本误收集？
- 是否区分开发版和安装版，避免互相污染？

不满足以上任一项，不允许合并。
